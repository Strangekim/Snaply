# ⏱ 지연 캡처(타이머) 명세

> 대상 버전: v1.5.1+ · 소유 코드: [src/main/capture/index.ts](../src/main/capture/index.ts) `startVisibleCountdown` / `finishDelayedCapture`, [src/renderer/src/overlay/App.tsx](../src/renderer/src/overlay/App.tsx)

## 1. 목적

드롭다운 메뉴, 우클릭 메뉴, 툴팁, 호버 효과처럼 **캡처를 시작하는 순간(포커스 이동) 사라지는 화면 상태**를 캡처하기 위한 기능. 캡처를 예약해 두고, 카운트다운 동안 사용자가 화면을 원하는 상태로 만들면 시간이 됐을 때 자동으로 찍는다.

## 2. 진입 방법

캡처 오버레이 상단 캡슐의 **⏱ 버튼 → 3초 / 5초 / 10초** 선택.

## 3. 동작 사양

타이머를 누른 시점의 상태에 따라 카운트다운 종료 시 동작이 달라진다.

| 타이머를 누를 때의 상태 | 카운트다운 종료(또는 Space) 시 |
|---|---|
| **영역이 지정돼 있음** (드래그 선택 또는 고정 크기 배치 후 조정 단계) | 그 영역을 **자동 캡처** — 오버레이 재진입 없음 |
| **스크롤 모드 + 영역 지정** | 그 영역에서 자동 스크롤 캡처 시작 |
| **전체 화면 모드** | **커서가 있는 모니터**를 자동 캡처 |
| 아무것도 지정하지 않음 (영역 모드에서 선택 전) | 캡처 오버레이가 다시 열림 — 화면을 세팅한 뒤 그때 영역을 고르는 용도 |

캡처 후 흐름(에디터 열기/클립보드)은 일반 캡처와 동일하게 설정의 "캡처 후 동작"을 따른다.

## 4. 카운트다운 중 화면 표시

- **카운트다운 배지**: 커서가 있는 모니터 우하단에 남은 초(3-2-1)와 안내 문구("Space 바로 캡처 · ESC 취소")를 다크 글래스 캡슐로 표시. 1초 간격 갱신.
- **캡처 예정 영역 포커스**: 자동 캡처될 영역(지정 영역 또는 전체 화면 모드의 커서 모니터 전체)에 **파란색 펄스 테두리**를 계속 표시한다. 영역이 다른 모니터에 있으면 그 모니터에 표시된다.
- 두 표시 모두:
  - `setContentProtection(true)` — **캡처 결과에 찍히지 않는다**
  - `setIgnoreMouseEvents(true)` — **클릭·타이핑이 아래 앱으로 통과**되어 카운트다운 중에 메뉴를 열거나 호버 상태를 만들 수 있다
  - 포커스를 뺏지 않는다(`showInactive`)

## 5. 카운트다운 중 키 입력

카운트다운 동안에만 전역 단축키가 등록되고, 종료 즉시 해제된다.

| 키 | 동작 |
|---|---|
| **Space** | 남은 시간과 무관하게 **즉시 캡처** — 원하는 순간이 만들어졌으면 기다릴 필요 없음 |
| **ESC** | 카운트다운 취소 — 아무것도 캡처하지 않고 종료 |

주의: 전역 등록이므로 카운트다운 동안에는 다른 앱에서도 Space/ESC 입력이 Snaply가 가로챈다 (최대 10초, 카운트다운 한정).

## 6. 대표 시나리오

**드롭다운 메뉴 캡처**
1. `PrintScreen` → 메뉴가 나타날 자리를 드래그로 영역 지정
2. ⏱ → 3초
3. 카운트다운 동안 대상 앱에서 드롭다운을 클릭해 연다 (클릭은 통과됨)
4. 메뉴가 열렸으면 **Space로 즉시 캡처** (또는 시간이 되길 기다림)

**툴팁/호버 상태**: 영역 지정 → ⏱ → 커서를 대상 위에 올려두고 대기 → 자동 캡처.

**화면 전체 + 준비 시간**: 캡슐에서 "전체 화면" → ⏱ → 창 배치 정리 → 커서 모니터 자동 캡처.

## 7. 구현 노트

- 진입: `capture:start` IPC에 `delayMs`(+ 선택적 `region`)가 오면 `startVisibleCountdown`이 실행된다. 오버레이의 조정 단계 선택 영역은 `RegionSelect.onSelChange` → `App.selectDelay`가 `region`으로 전달한다.
- 멀티 모니터: 오버레이 창은 디스플레이당 1개라 선택 영역과 타이머 캡슐이 서로 다른 창에 있을 수 있다. 각 창은 조정 단계 선택을 `overlay:selection`으로 메인에 보고하고, 타이머 시작 시 region이 비어 있으면 메인이 보관 중인 최신 선택으로 채운다 (핸드오프 순서 역전 대비 타임스탬프 기준).
- 종료 처리(`finishDelayedCapture`): region 있음 → 새 프레임 grab 후 크롭 저장 / 스크롤이면 자동 스크롤 캡처 / 전체 화면(displayId 미지정) → 커서 모니터 / 그 외 → `beginCapture` 재진입.
- 취소·완료 시 `cleanup`이 인터벌·전역 단축키·클릭 통과 상태·포커스 표시를 모두 원복한다. `capture:cancel`(트레이 등 외부 취소)도 `closeOverlay`를 통해 동일하게 정리된다.
- 관련 IPC 이벤트: `event:overlayCountdown`(남은 초), `event:overlayFocusRegion`(포커스 영역, 창 로컬 좌표).
- 검증: `tests/e2e/timed-capture.spec.ts`(IPC 경로), `tests/e2e/timed-ui.spec.ts`(실제 드래그→⏱→3초 UI 흐름).
